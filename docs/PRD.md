# PRD：AI 秋招投递决策助手

## 1. 项目背景

秋招过程中，学生需要同时处理大量岗位 JD、简历修改、投递进度和面试准备。传统做法依赖人工阅读 JD、手动判断匹配度、临时修改简历，效率低且难以复盘。

本项目希望把“岗位判断 + 简历优化 + 投递记录 + 数据复盘”整合到一个本地运行的 AI 辅助工具中，帮助用户更系统地完成秋招投递决策。

## 2. 目标用户

- 2027 届准备秋招或实习的本科生、硕士生。
- 目标岗位包括 AI 产品、产品经理、项目管理、数据分析、运营等。
- 希望提升投递效率，并将项目作为 AI 产品作品集展示的求职者。

## 3. 用户痛点

- JD 信息密度高，难以快速判断是否值得投递。
- 简历需要针对不同岗位调整，但容易变成机械堆关键词。
- 投递记录分散在 Excel、招聘平台和聊天记录里，后续复盘困难。
- AI 生成简历存在虚构经历风险，需要明确约束和人工确认。

## 4. 核心流程

1. 用户上传原始简历。
2. 用户粘贴岗位 JD，并填写公司、岗位、平台、城市、投递链接。
3. 系统解析简历文本。
4. 系统优先调用 LLM 输出结构化匹配分析；无 API Key 或调用失败时使用规则 fallback。
5. 系统保存投递记录和分析结果。
6. 用户生成针对岗位的定制简历草稿，并导出 Markdown / DOCX。
7. 用户在投递管理页更新状态、面试进展、下一步行动和面试备注。
8. 用户在数据看板页复盘投递方向、匹配度和 JD 高频能力要求。

## 5. MVP 功能

- 简历上传：支持 PDF、DOCX、TXT。
- 岗位匹配分析：输出匹配度、总体评价、匹配证据、优势、缺口、建议、风险。
- 定制简历生成：基于原简历和分析结果生成保守版 Markdown 简历。
- 投递记录管理：支持新增、筛选、编辑、删除、CSV 导出。
- 数据看板：展示投递数量、匹配度、状态分布、岗位类型分布、能力要求 Top 20 和投递趋势。

## 6. 页面结构

- 岗位匹配分析：上传简历、录入 JD、触发分析、展示结构化结果。
- 定制简历生成：展示免责声明、生成新简历、复制、导出 Markdown / DOCX。
- 投递记录管理：筛选、查看、更新状态与备注、导出 CSV。
- 数据看板：指标卡片和多维图表。

## 7. AI 能力设计

- LLM 输入：原始简历文本、岗位 JD。
- LLM 输出：结构化 JSON，便于页面展示和后续分析。
- Prompt 约束：只能基于用户上传的原始简历和 JD，不得虚构项目、公司、学校、奖项、指标。
- fallback 机制：无 API Key、调用失败或 JSON 解析失败时自动使用规则分析。

## 8. 指标设计

- 投递效率：总投递数量、最近 7 天 / 30 天投递趋势。
- 投递质量：平均匹配度、高匹配岗位数量、匹配度分布。
- 流程管理：待跟进岗位数量、状态分布、下一步行动。
- 方向复盘：岗位类型分布、不同岗位类型平均匹配度、JD 能力要求 Top 20。

## 9. 后续迭代

- 增加 LLM 简历生成链路，提升表达质量。
- 增加岗位收藏、优先级和投递决策理由。
- 增加更细粒度的面试复盘模块。
- 增加多版本简历管理和差异对比。
- 增加成本、延迟和模型调用成功率监控。
